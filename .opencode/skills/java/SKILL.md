# Java

Follow existing Java style and project conventions.

Check for:

- Null handling and Optional usage.
- Exception semantics.
- Transaction boundaries.
- Equality and hashing.
- Collection mutability.
- Stream readability.
- Concurrency and shared state.
- Test coverage for edge cases.

Prefer clear code over clever abstractions. Keep public APIs stable unless the user asks for an API change.
