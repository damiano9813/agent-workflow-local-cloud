# Feature Implementation

Use exploration before implementation.

1. Clarify behavior and constraints.
2. Ask the explore agent for read-only repository mapping when the change touches multiple files.
3. Validate relevant files, flow, dependencies, tests, and risks.
4. Implement the smallest maintainable solution.
5. Update tests and documentation when behavior changes.
6. Ask the reviewer agent for read-only review when useful.
7. Run verification and summarize residual risks.

Avoid speculative abstractions and broad rewrites.
