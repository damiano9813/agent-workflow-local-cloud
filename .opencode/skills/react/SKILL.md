# React

Before changing frontend behavior, inspect relevant:

- Components.
- Custom hooks.
- State management.
- Context providers.
- React Query usage.
- API clients.
- Forms.
- Validation.
- Routing.
- Frontend tests.

Preserve the existing design system and project conventions. Prefer simple component changes over new abstractions.

Verify desktop and mobile behavior when UI layout changes.
